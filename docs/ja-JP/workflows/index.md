# ワークフロー概要

ZCFはMCP + ワークフローテンプレートを通じて、チームの開発プロセスを標準化するのに役立ちます。この章では、各ワークフローの位置づけと使用方法を紹介します。

| ワークフロー | 適用シナリオ | 主要機能 |
| --- | --- | --- |
| [ZCF 6段階ワークフロー](zcf-workflow.md) | 汎用開発タスク | 6段階の閉ループ、自動品質チェック、対話式確認 |
| [機能開発ワークフロー](feat.md) | 新機能の設計と実装 | 計画 + UI/UXエージェント連携 |
| [BMad アジャイルプロセス](bmad.md) | 大規模プロジェクトのアジャイル反復 | マルチステージセレモニー管理 |
| [Spec ワークフロー統合](spec.md) | 要件ドキュメントと仕様生成 | Spec MCP統合 |
| [Git スマートコマンド](git-commands.md) | Git操作の自動化 | コミット、ロールバック、クリーンアップ、worktree |

## 使用方法

使用するツールに応じて、異なるコマンドプレフィックスを使用します：

### Claude Code

Claude Codeで以下のコマンドを使用：
- `/zcf:workflow` - 6段階ワークフロー
- `/zcf:feat` - 機能開発ワークフロー
- `/git-commit` - Gitコミットコマンド
- `/init-project` - プロジェクト初期化

### Codex

Codexで以下のコマンドを使用（`/prompts:`プレフィックスに注意）：
- `/prompts:workflow` - 6段階ワークフロー
- `/prompts:git-commit` - Gitコミットコマンド
- `/prompts:git-rollback` - Gitロールバックコマンド
- `/prompts:git-cleanBranches` - マージ済みブランチのクリーンアップ
- `/prompts:git-worktree` - Git worktree管理

> ⚠️ **注意**: Codexは現在、6段階ワークフローとGitワークフローのみをサポートしています。機能開発ワークフロー（feat）、プロジェクト初期化（init-project）、BMadワークフローはCodexではまだ利用できません。

> 💡 **ヒント**: Codexはすべてのワークフローコマンドにアクセスするために`/prompts:`プレフィックスを使用します。これはCodexのコマンド形式仕様です。

## 使用推奨事項

1. 初めてワークフローを使用する際は、AIにタスク進捗ドキュメントを出力させ、新しい会話で簡単に続行できるようにします。
2. 重要なマイルストーンを完了した後、AIに進捗サマリーの生成を依頼し、会話間の連続性を確保します。
3. `best-practices/worktree.md`と組み合わせて、複数ワークスペースのコラボレーション効率を向上させます。
4. Gitワークフローと組み合わせて使用すると、要件の分解→コーディング→コミットのサイクルを迅速に完了できます。
